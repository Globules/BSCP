## File path traversal, traversal sequences stripped non-recursively

This lab contains a path traversal vulnerability in the display of product images.

The application strips path traversal sequences from the user-supplied filename before using it.

To solve the lab, retrieve the contents of the /etc/passwd file.

## Notes

- /image?filename=X

## Resume

1. Access to any product
2. Access image url (right click on image, send to new tab)
3. Update the url to read `/etc/passwd`

## Solve 

First step is to go on any product : `/product?productId=1` then right click on the product image and open it on a new tab : `/image?filename=25.jpg`.

Update the previous url to read **/etc/passwd** : `/image?filename=../../../../etc/passwd` -> "No such file"

We have to find a way to bypass and make our traversal sequences work. In order to do this we can double our traversal sequences. If the first are filtered and not the second one we should be able to read any file. 

`....//....//....//etc//passwd` -> This will display the file content we're accessing. 

If we use the direct url, we can't read the **/etc/passwd** it only display images. To get the file content we just need to replay the request in Burp :
